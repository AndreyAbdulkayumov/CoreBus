# Локализация CoreBus

Каждый файл `*.json` в этой папке — один язык интерфейса приложения.

Чтобы **добавить новый язык**, достаточно:

1. Скопировать `en.json` в файл с нужным кодом языка, например `de.json`.
2. Заполнить блок `_meta` (см. ниже) и перевести значения.
3. Пересобрать проект (файлы копируются в выходной каталог автоматически благодаря
   правилу в `CoreBus.Base.csproj`) — **или просто положить файл рядом с исполняемым
   файлом в уже собранном приложении**, в той же подпапке `Localization`.

Код менять не нужно: сервис локализации (`LocalizationService`) сканирует папку
`Localization/` рядом с исполняемым файлом и подхватывает любые новые `*.json`.

## Формат файла

```json
{
  "_meta": {
    "code": "en",
    "name": "English",
    "englishName": "English"
  },

  "Common": {
    "Close": "Close",
    "Save": "Save"
  },

  "About": {
    "Developer": "Developed by:"
  }
}
```

- `_meta.code` — короткий код языка, по которому выбор сохраняется в настройках
  (`AppInfo.LanguageCode`). Если поле не указано, используется имя файла без
  расширения.
- `_meta.name` — отображаемое имя языка в выпадающем списке настроек (обычно на
  самом языке: «Русский», «English», «Deutsch»).
- `_meta.englishName` — тот же язык на английском, для логов и диагностики.
- Все остальные поля — иерархический словарь переводов. В XAML и коде
  используются плоские ключи через точку: `"Common.Close"`, `"About.Developer"`.

## Использование в XAML

В корне `.axaml` объявите пространство имён один раз:

```xml
xmlns:l="clr-namespace:CoreBus.Base.Localization;assembly=CoreBus.Base"
```

И используйте:

```xml
<TextBlock Text="{l:Loc Common.Close}"/>
<Button ToolTip.Tip="{l:Loc About.CopyEmail}"/>
```

При смене языка все привязки автоматически обновятся — перезапуск не нужен.

## Использование в C# (ViewModels)

Внедряйте `ILocalizationService` через DI и обращайтесь так:

```csharp
_messageBox.Show(_localization.Get("Error.ThemeSwitchDark"), MessageType.Error);

// с форматированием:
var msg = _localization.Get("Info.NewVersionAvailable", version);
```

## Fallback

- Если ключа нет в файле текущего языка — возвращается сам ключ
  (удобно для диагностики: сразу видно, где потерян перевод).
- Если сохранённый язык отсутствует — используется `ru`, а если и `ru` нет —
  первый доступный язык из папки.
