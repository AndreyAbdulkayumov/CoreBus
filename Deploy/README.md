# Публикация приложения

1. Внести необходимые изменения в проект.

1. Дополнить документацию:
	* Поменять версию в титульном листе.
	* При необходимости описать новый функционал.
	* Добавить описание изменений в разделе "История версий".	
	* Внедрить обновленную документацию в проект (см. README.md в папке `CoreBus\Documentation`).

1. В файле проекта `CoreBus.Base` изменить версию в теге `Project\PropertyGroup\Version`. Собрать проект и проверить номер версии в окне "О приложении".

1. В случае изменения версии .NET поменять значения в тегах `Project\PropertyGroup\TargetFramework` во всех файлах проектов.

1. Опубликовать проект `CoreBus.Desktop` для Windows и Linux. Использовать только эти файлы для дальнейшего распространения!

1. Создать версию с установщиком с помощью скрипта.

1. Создать portable-версии:
	* Удалить .pdb-файлы в опубликованных папках.
	* Переименовать папки согласно следующему шаблону (x.x.x - номер версии):

		```
		CoreBus_x.x.x_linux_x64
	    ```
	    ```
		CoreBus_x.x.x_win_x64_portable
	    ```

	* Создать .zip-архивы с этими папками внутри. Названия архивов должны совпадать с названиями папок.
	
   Создать архив для Linux можно только в дистрибутиве Linux. 
   
   Обязательно проверить содержимое архивов!

1. Выложить приложение на Яндекс.Диск. Для каждого файла создать свою папку. Названия папок идентичны названию файлов внутри.
	* CoreBus_x.x.x_linux_x64
	* CoreBus_x.x.x_win_x64_installer
	* CoreBus_x.x.x_win_x64_portable

1. Обновить сайт приложения. 
	* Обновить версию в `lastestVersion.json`.
	* Дополнить страницу загрузок.

1. Выложить приложение на остальных ресурсах.
